Comandos básicos de la terminal

pwd
Devuelve la ruta del directorio en que estás ubicado.
ls
Devuelve los archivos y carpetas que hay en el directorio en el que estás ubicado.
ls -al
Devuelve los archivos y carpetas que hay en el directorio en el que estás ubicado, incluyendo los archivos ocultos.
ls -a
Muestra los archivos ocultos dentro de un directorio.
ls -l
Muestra los archivos en forma de lista en un directorio.
mkdir
Crea un nuevo directorio o directorios en la ubicación actual. Se debe agregar el nombre del directorio. Ejemplo: mkdir nombre-de-la-carpeta.
touch
Crea un nuevo archivo o archivos en la ubicación actual. Se debe agregar el nombre del archivo más su extensión. Ejemplo: touch archivo.txt.
rm
Elimina archivos de un directorio. Se debe agregar el nombre del archivo y su extensión. Ejemplo: rm archivo.txt.
rm -d
Elimina un directorio vacío. Se debe agregar el nombre del directorio. Ejemplo: rm -d nombre-del-directorio-vacío.
rm -r
Elimina un directorio que contiene archivos. Se debe agregar el nombre del directorio. Ejemplo: rm -r nombre-del-directorio.
rm -ri
Elimina un directorio que contiene archivos. Se debe agregar el nombre del directorio. Ejemplo: rm -ri nombre-del-directorio. La terminal te preguntará si estás seguro de eliminar los archivos o carpetas y deberás seleccionar sí o no.
head
Muestra las primeras 10 líneas de un archivo de texto.
head -n
Muestra las primeras líneas que indiques de un archivo de texto. Ejemplo: head -n 5 archivo.txt muestra las primeras 5 líneas.
tail
Muestra las últimas 10 líneas de un archivo de texto.
tail -n
Muestra las últimas líneas que indiques de un archivo de texto. Ejemplo: tail -n 5 archivo.txt muestra las últimas 5 líneas.
less
Muestra el contenido de un archivo de texto, permitiendo desplazarse por él.
cat
Muestra el contenido de un archivo específico. Se debe agregar el nombre del archivo y su extensión. Ejemplo: cat archivo.txt.
cd
Permite ingresar a una carpeta que esté dentro del directorio. Se debe agregar el nombre de la carpeta. Ejemplo: cd nombre-de-la-carpeta.
cd ..
Permite volver a un nivel anterior de la carpeta en la que te encuentras.
history
Muestra el historial de todos los comandos que se han ejecutado.
! + número
Ejecuta el comando asignado a un número en el historial.
clear
Limpia la consola.
type
Muestra el tipo de comando.
man
Muestra la información que el manual tiene del comando.
info
Muestra información general del comando.
whatis
Muestra una información bastante breve del comando.

Comandos básicos de Git

git init
Inicia un repositorio en el directorio actual.
git status
Muestra el estado del repositorio.
git add .
Agrega los archivos del directorio al stage y los prepara para el commit.
git commit -m
Agrega los archivos al repositorio, dejando una versión de los archivos en el historial.
git commit -am
Permite agregar los cambios de un archivo o archivos en el repositorio y hacer un commit o versión a la vez.
git log
Muestra todo el historial de los commits o versiones realizadas de los archivos del directorio.
git reset soft
Elimina los archivos del repositorio, pero no los elimina del stage ni del directorio.
git reset mixed
Elimina los archivos del repositorio y del stage, pero no del directorio.
git reset hard
Elimina los archivos del repositorio, del stage y del directorio.
git rm
Elimina los archivos del directorio, pero no elimina las versiones del repositorio.
git checkout
Permite cambiar entre las versiones que existen en el repositorio y entre las ramas existentes.
git branch
Permite crear una nueva rama. Se debe agregar el nombre de la nueva rama.
git show-branch
Muestra todas las ramas que existen en forma de lista.
gitk
Muestra una interfaz gráfica de Git.
git checkout -b
Genera una nueva rama y nos mueve a ella directamente.
git reset id-commit
Nos lleva a cualquier commit, sin importar la rama, eliminando el historial de los commits posteriores al tag seleccionado.
git reset rama-o-id-commit
Nos lleva a cualquier commit sin borrar los commits posteriores al tag seleccionado.

Comandos básicos de Git con GitHub

git remote add origin
Agrega un origen remoto donde se ubicarán los repositorios o se copiará un repositorio. Se debe agregar el enlace proporcionado por GitHub.
git remote
Muestra si tenemos un origen remoto configurado.
git remote -v
Muestra los tipos de orígenes que tenemos, usualmente fetch y push.
git branch -m main
Establece la rama main como principal.
git push origin main
Sube nuestro repositorio local al origen remoto.
git pull origin main
Copia el repositorio remoto a nuestro repositorio local o lo actualiza en caso de que ya haya sido copiado anteriormente.
--allow-unrelated-histories
Permite fusionar historias sin relación al copiar un repositorio remoto en nuestro local. Se debe usar con git pull.
git log --all --graph --decorate --oneline
Muestra los commits con un pequeño diagrama y la información resumida.
git tag -a v0.n -m
Agrega un tag con el nombre v0n.
git tag
Muestra los tags existentes.
git show-ref --tags
Muestra los tags junto con sus commits.
git push origin --tags
Sube todos los tags al repositorio remoto.
git tag -d nombre
Elimina el tag del repositorio local.
git push origin :refs/tags/nombre
Elimina el tag del repositorio remoto.
